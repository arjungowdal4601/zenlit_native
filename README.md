# Zenlit Native

Zenlit is an Expo React Native app for location-first networking. The main app entry is Radar, with supporting Feed, Create, Chat, and Profile screens. Authentication uses Supabase email OTP, public profile data lives in Supabase, and onboarding is guarded by a backend-driven resolver before the bottom tabs are shown.

## Quick Start

Prerequisites:

- Node.js and npm
- Expo CLI through `npx expo`
- A Supabase project with email OTP enabled
- Expo Go or a development build for device testing

Install dependencies:

```bash
npm install
```

Create a local `.env` file:

```bash
EXPO_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
EXPO_PUBLIC_SUPABASE_ANON_KEY=your-anon-or-publishable-key
```

Only Supabase anon/publishable keys belong in Expo public environment variables. Do not put service-role keys or other secrets in `.env` values that start with `EXPO_PUBLIC_`.

Run the app:

```bash
npm start
npm run ios
npm run android
npm run web
```

`npm run dev` starts the web target and is useful for quick UI checks.

## Onboarding Summary

Authenticated routing is intentionally centralized. `src/services/onboardingService.ts` reads Supabase profile state, `src/utils/onboardingState.ts` evaluates it, and `src/utils/authNavigation.ts` exposes route helpers used by the root layout and onboarding screens.

The required journey is:

```text
Get Started -> Email OTP -> Profile Basics -> Optional Profile Details -> Radar
```

Required profile basics are display name, username, date of birth, and gender. Optional details are avatar, banner, bio, Instagram, X, and LinkedIn. Optional details never block Radar once required basics are valid.

## Supabase Summary

The connected remote public schema currently exposes:

- Tables: `profiles`, `social_links`, `posts`, `feedback`, `locations`, `messages`
- View: `locations_current`
- RPCs: chat authorization, email/username availability, username suggestions, location expiry, unread counts, and message delivery/read markers
- Storage buckets from local migrations and app usage: `profile-images`, `post-images`, `feedback-images`
- Edge functions in source: `send-push-notification`, `update-conversation-anonymity`

Local files include the onboarding draft migration `supabase/migrations/20260520090000_create_profile_basics_drafts.sql`. The connected remote migration list does not currently show that migration.

`profile_basics_drafts` is required for reinstall-safe onboarding drafts and must be applied before relying on that behavior remotely.

Do not apply Supabase migrations as part of documentation or test-only work. Treat remote state and local migration files as separate facts until a backend migration task is approved.

## Testing

The test foundation uses Jest with `jest-expo` and React Native Testing Library. Tests live outside `app/`:

- `test/unit/` for pure routing, validation, and parsing logic
- `test/ui/` for screen/component/router behavior with mocked native modules
- `test/setup/jest.setup.tsx` for native, Expo, and Supabase test doubles
- `test/utils/render.tsx` for shared render helpers

Useful commands:

```bash
npm run typecheck
npm run test:unit
npm run test:ui
npm test
npm run build
```

More details are in `docs/knowledge-base/testing.md`.

## Common Development Workflow

1. Pull the latest branch and run `npm install`.
2. Add or update `.env` with local Supabase public values.
3. Make scoped app changes in `app/`, `src/`, or `supabase/`.
4. Add unit tests for pure logic and UI tests for high-risk screens.
5. Run `npm run typecheck`, targeted tests, then `npm run verify` before handing off.
6. If backend structure changes, update `supabase/migrations/`, generated types, and `docs/knowledge-base/backend.md` in the same task.

For a deeper orientation, start with:

- `docs/knowledge-base/architecture.md`
- `docs/knowledge-base/backend.md`
- `docs/knowledge-base/testing.md`
