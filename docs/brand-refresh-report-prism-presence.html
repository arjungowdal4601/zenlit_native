Prism Presence brand direction

Zenlit as a premium presence system.

Prism Presence keeps Zenlit dark and social, but makes it feel sharper, cleaner, and more premium. It uses blue and violet as the brand core, with cyan as a precise signal highlight for radar, visibility, focus, and active system feedback.

Best For

A premium social discovery app that should feel calm, private, accurate, and high-trust.

Main Risk

If overused, cyan can make the app feel cold or technical instead of human.

Recommendation

Use Carefully

Use Prism Presence only if you want Zenlit to feel more premium and precise than warm and social.

Executive Recommendation

  • Brand direction: Prism Presence is the strongest premium option, but not the safest default.
  • Use case: best if Zenlit should feel like a refined radar/presence product, not a casual social app.
  • Typography: Satoshi for brand/display, Geist for UI text. This feels sharper than Sora + Inter.
  • Color: blue and violet stay primary; cyan becomes the signal/focus accent.
  • Implementation: still start with tokens and core flow only. Do not repaint every screen at once.

Visual Direction Board

How Prism Presence Differs

Compared To Signal Noir

Signal Noir is more familiar and slightly warmer. Prism Presence is cleaner, sharper, and more premium.

Compared To Nearlight

Nearlight feels more human and inviting. Prism Presence feels more composed, controlled, and product-led.

Best Product Message

“Presence, nearby.” Short, direct, and aligned with the radar/visibility concept.

Typography Recommendation

Zenlit

Satoshi for brand moments

Satoshi gives Prism Presence a premium, editorial feel. Use it for the wordmark, hero-scale text, onboarding titles, and brand-led section titles.

Geist is cleaner for UI forms, helper text, buttons, and dense screens. It feels precise without becoming decorative.

Type Rules

  • Use Satoshi only for brand/display moments.
  • Use Geist for all product UI and forms.
  • Keep letter spacing at 0 except tiny eyebrow labels.
  • Use fewer weights: regular, medium, semibold, bold.

UI System Changes

Prism Components

  • ScreenShell: black/prism background, status bar, and safe-area consistency.
  • ZenlitButton: primary blue-violet-cyan gradient, secondary cyan outline.
  • ZenlitCard: darker glass-like card, thin steel border, radius 12.
  • ZenlitInput: quiet dark input, cyan focus, green success, red error.
  • RadarMark: thin radar rings and prism beam, not decorative blobs.

Where I Would Apply It First

  • Get Started: central prism/radar motif, brand wordmark, one clean CTA.
  • Auth and OTP: refined card, consistent input, cyan focus state, same CTA treatment.
  • Profile Basics and Complete Profile: same onboarding shell and step progress style.
  • Recovery: use the same shell so error handling does not feel like a different product.
  • Radar: make cyan represent active radar focus, not every button or label.
  • Edit Profile shell: reuse the Complete Profile media/header treatment.

What To Be Careful About

  • Do not overuse cyan. It should mean signal, focus, radar, or active state.
  • Do not make every surface glossy. Keep the UI fast, readable, and simple.
  • Do not introduce huge gradients in backgrounds. Keep gradients for text, CTAs, and controlled highlights.
  • Do not implement Prism across every screen until the core flow feels right.
  • Do not let the brand become cold. Keep copy human and direct.
