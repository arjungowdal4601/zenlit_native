Branding and UI consistency report

Zenlit should feel nearby, private, alive.

My recommendation is not a full rebrand. Keep the current black, electric blue, and violet foundation, then make it ownable with one brand token system, a restrained presence accent, sharper typography, and a reusable radar/signal visual language.

Problem

The app currently looks generic because colors, typography, cards, headers, and buttons are defined differently screen by screen.

Best Fix

Centralize brand tokens and migrate the first-run product path before touching every page.

Recommendation

Use the Signal Noir direction: current Zenlit DNA, stronger radar/presence motif, and a tiny amber accent for “nearby now”.

Executive Recommendation

  • Brand direction: Signal Noir. It keeps Zenlit recognizable but makes it feel designed.
  • Typography: Sora for Zenlit/headline moments, Inter or system fallback for product UI.
  • Color: keep black + blue/violet, add amber only for live presence and “nearby now” moments.
  • Visual language: subtle radar rings, signal lines, presence dots, and privacy shield cues.
  • Implementation: core flow first: Get Started, Auth, OTP, Profile Basics, Complete Profile, Recovery, Radar, and Edit Profile shell.

Direction Options

Recommended

Typography Recommendation

Zenlit

Sora for presence moments

Sora gives the brand a more ownable shape than default system fonts. Use it for Zenlit, onboarding titles, and high-emotion moments like “Set up your presence.”

Inter remains the practical UI font for inputs, forms, helper text, and repeated operational screens.

Type Rules

  • Do not hardcode font sizes per screen unless the token is missing.
  • Use display, title, body, label, caption, and button text tokens.
  • Keep letter spacing at 0 for most UI text.
  • Use weight and color together carefully; avoid making everything bold.

UI System Changes I Would Make

Shared Primitives

  • ScreenShell: one background, safe-area, and status bar pattern.
  • ZenlitButton: one primary/secondary/destructive/loading treatment.
  • ZenlitCard: one surface, border, shadow, and radius language.
  • ZenlitTextInput: one input, error, success, disabled, and helper pattern.
  • BrandText: typography tokens instead of scattered font sizes.
  • RadarMark: subtle reusable radar/signal motif for brand moments.

Where To Apply First

  • Get Started: remove generic feel, add radar/signal motif, use Sora brand wordmark.
  • Auth and OTP: same card, input, button, helper text, and legal link style.
  • Profile Basics and Complete Profile: same onboarding shell, same step labels, same buttons.
  • Recovery: make it feel like the same auth/onboarding system, not a separate error page.
  • Radar: make the radar motif a product surface, not only a button label.
  • Edit Profile shell: use the same header and profile media treatment as Complete Profile.

What Not To Do

  • Do not make a huge rebrand before the product flow is stable.
  • Do not add more random gradients; make the existing gradient disciplined.
  • Do not use warm amber everywhere; reserve it for presence/live/nearby cues.
  • Do not refactor every screen in one pass; start with the user’s first-run path.
  • Do not turn this into a marketing landing page. Zenlit needs the app experience first.
